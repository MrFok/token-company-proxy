Logging instructions for token-company-proxy
Reference sources:
- https://loggingsucks.com/
- OpenTelemetry logging concepts and semantic conventions

1) Core logging philosophy
- Emit structured JSON logs only.
- Prefer a canonical wide event per request completion.
- Log request outcomes and context, not step-by-step noise.
- Keep logs queryable and correlated by request_id/trace_id.

2) Required fields on canonical request events
- timestamp
- level
- event_name
- request_id
- trace_id (if available)
- service
- method
- path
- status_code
- duration_ms
- outcome
- upstream_base_url (sanitized)
- upstream_status_code

3) Compression fields for request events
- compression.enabled
- compression.eligible
- compression.attempted_count
- compression.applied_count
- compression.fallback_count
- compression.skipped_count
- compression.reason_codes
- compression.model
- compression.aggressiveness
- compression.input_chars_before
- compression.input_chars_after
- compression.reduction_pct

4) Security and privacy requirements
- Never log API keys, bearer tokens, cookies, or auth headers.
- Never log raw prompt content by default.
- Redact known key patterns and secret-like values.
- If debug body logging is ever added, it must be opt-in and truncated.

5) Log levels
- info: normal lifecycle events and request completion
- warn: fallbacks, retries, degraded paths
- error: failures and exceptions
- debug: local troubleshooting only

6) Local log access endpoint
- Local endpoint: GET /debug/logs
- Must be disabled in production.
- Keep an in-memory ring buffer with sanitized structured events.
- Support filtering: level, request_id, since, limit.

7) Sampling guidance
- Keep 100% of error/warn events.
- Keep 100% of compression fallback events.
- Allow sampling of high-volume successful events in hosted environments.
- Keep full logging in local development.

8) Correlation and consistency
- Include x-proxy-request-id in responses.
- Carry trace_id when present in incoming headers.
- Keep field naming consistent across logs and /stats.

9) Canonical event names
- proxy.server.started
- proxy.request.completed
- proxy.request.failed
- proxy.compression.fallback

10) Validation checklist
- A failing request is diagnosable by request_id with one query.
- Logs contain no secrets under normal operation.
- Compression decisions are visible per request.
- /debug/logs is available only in local mode.
